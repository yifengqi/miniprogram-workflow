## 一、项目基本信息【已明确】

### 1.1 项目背景

```
核心痛点：
每天早上站在衣柜前 20 分钟不知道穿什么，衣服买了一堆但永远觉得"没衣服穿"。

具体表现：
1. 决策疲劳 —— 每天在"穿什么"上浪费大量时间和精力
2. 搭配盲区 —— 衣服不少但不会搭，总是穿固定几套
3. 重复尴尬 —— 同一周在同事/朋友面前穿重复的衣服
4. 盲目购物 —— 不清楚衣橱缺什么，凭感觉买，导致同类单品过剩
5. 换季混乱 —— 不知道哪些衣服该淘汰，衣柜越来越满
```

## 二、用户相关【已明确】

### 2.1 主要用户是谁？

```
核心用户画像：

画像A - 都市白领女性（主力用户，占比约60%）
  · 年龄：22-35岁
  · 特征：工作日需要通勤穿搭，周末有社交/约会/逛街等不同场景
  · 痛点：早上赶时间、希望穿得体面但没时间搭配
  · 衣服数量：50-150件

画像B - 职场新人（占比约20%）
  · 年龄：20-25岁
  · 特征：刚入职不久，正在建立个人衣橱，对穿搭缺乏经验
  · 痛点：不知道怎么搭、不知道该买什么
  · 衣服数量：30-80件

画像C - 时尚爱好者（占比约15%）
  · 年龄：18-40岁
  · 特征：衣服很多，喜欢尝试新搭配，关注时尚趋势
  · 痛点：衣服太多记不住有什么，想要更系统地管理
  · 衣服数量：100-300件

画像D - 注重效率的男性用户（占比约5%，后续可扩展）
  · 年龄：25-40岁
  · 特征：不想花时间在穿搭上，但希望穿得得体
  · 痛点：穿搭知识少、想要最省心的方案
```

### 2.2 用户使用场景

```
场景1 - 每日穿搭决策（高频，核心场景）
  · 时间：早上7:00-8:30
  · 触发：起床后打开小程序，查看AI推荐的今日搭配
  · 期望：30秒内看到适合今天天气和日程的穿搭方案

场景2 - 衣物录入（中频，必要场景）
  · 时间：购物回家后 / 周末整理衣柜时
  · 触发：新买了衣服，拍照录入系统
  · 期望：拍照后AI自动识别类别、颜色、风格，减少手动填写

场景3 - 穿搭记录（中频）
  · 时间：出门前
  · 触发：确认今天穿的搭配，标记"今日穿搭"
  · 期望：一键记录，积累穿搭数据

场景4 - 购物参考（低频但高价值）
  · 时间：逛商场/线上购物前
  · 触发：查看"衣橱缺什么"功能
  · 期望：得到有理有据的购物建议，避免冲动消费

场景5 - 朋友圈穿搭分享（中高频，社交场景）
  · 时间：出门前/穿搭确认后
  · 触发：确认今日穿搭后，想在朋友圈秀一下
  · 期望：一键生成好看的穿搭卡片/海报，配上有趣的AI文案，直接发朋友圈
  · 情绪价值：获得朋友点赞、被夸"会穿"、成为朋友圈穿搭博主

场景6 - 换季整理（季度性）
  · 时间：季节更替时
  · 触发：换季提醒推送
  · 期望：清楚知道哪些可以淘汰、哪些需要收纳
```

## 三、核心功能【已明确】

### 3.1 必须要有的功能（P0）

```
1. 衣物拍照录入 + AI智能识别
   - 拍照/相册选图录入衣物
   - AI自动识别：类别（上衣/裤子/裙子/鞋/包/配饰）、颜色、风格
   - 支持手动修正和补充信息（品牌、购入时间、价格等）
   - 衣橱分类浏览（按类别、颜色、季节、场合）

2. AI智能穿搭推荐
   - 获取当日天气信息（温度、天气状况）
   - 用户设置今日场合（通勤/约会/见客户/周末逛街/运动/聚餐等）
   - AI根据天气+场合+个人衣橱，推荐2-3套今日搭配
   - 搭配展示：上衣+下装+鞋+包+配饰的完整look
   - 用户可换单品（"上衣换一件"）

3. 穿搭日历 + 重复检测
   - 日历形式记录每天穿了什么
   - 确认今日穿搭后自动记录
   - 智能检测：同一周内是否有重复穿搭
   - 重复预警："这套搭配你周二刚穿过，建议换一下"
```

### 3.2 希望有的功能（P1）

```
1. 衣橱缺口分析
   - AI分析衣橱结构（品类占比、颜色分布、场合覆盖度）
   - 识别缺失单品："你有5条裙子但没有一双百搭的裸色高跟鞋"
   - 给出购物建议清单（优先级排序）
   - 预算友好：按价格区间推荐

2. 换季管理
   - 根据当前季节智能提醒换季整理
   - 分析衣物使用频率："这件外套过去一年只穿了1次"
   - 淘汰建议 + 理由（过时/损耗/不合身/一年未穿）
   - 收纳指导：哪些当季、哪些收起来

3. 衣物详情与统计
   - 每件衣物的穿着次数、最后穿着日期
   - 单次穿着成本（价格 ÷ 穿着次数）
   - 衣橱总资产统计
   - 使用率排行榜：最爱穿 / 最少穿

4. 天气自动获取
   - 基于定位自动获取当日天气
   - 未来3天天气预览（提前规划穿搭）
   - 天气骤变提醒："明天降温10°，建议准备外套"

5. 场合标签管理
   - 预设场合：通勤、约会、见客户、周末休闲、运动、聚餐、旅行
   - 支持自定义场合
   - 日程联动（可选）：绑定手机日历自动获取今日场合

6. 朋友圈内容工厂（穿搭社交分享）
   - 一键生成精美穿搭卡片/海报
     · 多种模板风格：杂志风、极简风、ins风、文艺风、可爱风
     · 自动排版：搭配照片 + 天气 + 日期 + 心情标签
     · 支持自定义背景色/贴纸/滤镜
   - AI自动生成朋友圈文案
     · 风格可选：文艺范、搞笑吐槽、简约高级、emoji派对、毒鸡汤
     · 示例："今天是人间富贵花本花 🌺 28°C的天气配碎花裙刚刚好"
     · 示例："第47天不重样穿搭打卡 | 衣橱里的每件衣服都不被辜负"
     · 示例："会议穿搭battle冠军已就位 💼 老板看了都想给我加薪"
     · 一键复制文案，粘贴到朋友圈
   - 穿搭数据卡片（趣味社交货币）
     · "本月穿搭报告"：最爱颜色、最常风格、搭配多样性评分
     · "衣橱人格"：AI根据穿搭习惯生成趣味人格标签（如"极简主义者""色彩猎手""混搭女王"）
     · "穿搭成就"：连续N天不重样、解锁全部品类、单品穿着100次等里程碑
     · "单品性价比王"：这件T恤穿了52次，每次只花0.5元
   - 一键分享到微信朋友圈 / 好友 / 收藏
   - 保存高清图片到相册
```

### 3.3 锦上添花的功能（P2）

```
1. 穿搭社区分享（站内社区，P1朋友圈分享的延伸）
   - 分享今日穿搭卡片到站内社区
   - 浏览他人穿搭获取灵感
   - 点赞、收藏搭配方案
   - 热门穿搭排行榜

2. 节日/热点限定模板
   - 节日主题卡片（春节红、七夕粉、圣诞绿...）
   - 热点话题模板（秋天的第一杯奶茶配什么穿搭、年会战袍...）
   - 季节限定："入秋第一件外套""夏日清凉穿搭"
   - AI追热点文案：自动结合当下流行梗

3. 搭配收藏夹
   - 收藏满意的搭配方案
   - 按场合/季节分类收藏
   - 快速复用历史搭配

4. 虚拟试衣
   - 将衣物照片拼合预览搭配效果
   - 简单的上衣+下装+鞋拼图展示

5. 购物清单同步
   - 从"衣橱缺口分析"一键生成购物清单
   - 支持标记已购买
   - 购物链接推荐（如对接电商平台，需后续讨论商业模式）

6. 穿搭AI助手对话
   - 自然语言交互："周五约会穿什么好？"
   - 风格诊断："我适合什么风格？"
   - 搭配教学："怎么把这件碎花裙穿出高级感？"
```

### 10.2 其他需要说明的事项

```
关键技术挑战：
1. 衣物图像识别准确率 —— 需要AI能准确识别衣物类别、颜色、风格
2. 穿搭推荐算法质量 —— 推荐的搭配需要真正好看、合理
3. 照片存储成本控制 —— 每个用户可能上传50-300张衣物照片
4. 冷启动问题 —— 新用户需要先录入足够的衣物才能体验AI推荐

应对策略：
1. 图像识别：先用成熟的AI视觉API（GPT-4V / 通义千问VL），后续可训练专用模型
2. 推荐算法：结合色彩搭配理论 + 时尚规则 + 用户反馈持续优化
3. 存储优化：图片压缩 + 缩略图策略 + 超大图裁剪
4. 冷启动：提供批量拍照模式，快速录入；同时提供通用穿搭建议

隐私安全考虑：
1. 衣物照片仅存储在用户自己的云空间
2. AI分析在服务端完成，不存储原始数据
3. 社区分享需用户主动授权
4. 支持一键清空所有个人数据
```
