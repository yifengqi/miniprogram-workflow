> 一个基于真实项目经验（星见StelRen小程序）提炼的标准化开发流程工具
> 专为"零经验客户+AI协作开发"场景设计

**文档版本**：v1.0  
**创建时间**：2026-02-06  
**项目性质**：开发流程工具 + 经验知识库

## 🎯 一、项目概览

### 1.1 项目定位

这是一个**帮助开发者和AI更好协作的标准化工作流工具**，不是一个具体的业务小程序，而是：

- ✅ **开发流程框架**：从需求收集到上线的完整流程
- ✅ **PRD生成工具**：AI辅助生成客户版和开发版PRD文档
- ✅ **经验知识库**：记录和积累开发问题与解决方案
- ✅ **质量检查清单**：非功能性需求的系统化检查
- ✅ **数据管理系统**：支持项目数据的导出导入和备份

### 1.2 核心价值

```
传统开发流程的痛点                    本工具的解决方案
├─ 需求沟通反复不清楚      →      结构化需求收集表单
├─ 客户看不懂技术文档      →      双版本PRD（客户版+开发版）
├─ 开发过程混乱无序        →      标准化的阶段划分和检查点
├─ 非功能性需求常遗漏      →      系统化的检查清单
├─ 经验无法积累传承        →      可持续更新的知识库
└─ 重复踩相同的坑          →      AI自动读取历史问题库
```

### 1.3 适用场景

**最适合的场景**：
- 开发者 + AI 协作开发小程序
- 客户零技术背景，需要简单易懂的沟通方式
- 希望建立标准化流程，提高开发效率
- 需要积累经验，避免重复踩坑

**不适合的场景**：
- 大型团队的复杂项目（本工具面向小型项目）
- 完全不用AI的纯人工开发（工具为AI协作优化）

## 📁 三、标准化开发流程

### 3.1 完整流程概览

```
┌──────────────────────────────────────────────────────────────┐
│                     阶段零：知识库加载                           │
│  AI自动读取【07_经验知识库】，了解历史项目问题和解决方案           │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                     阶段一：需求收集                            │
│  1. 使用【01_需求收集模板】向客户提问                            │
│  2. 客户填写需求，整理记录                                      │
│  3. 将整理好的需求提供给AI                                      │
│  ⏱️ 参考时间：1-2小时                                          │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                     阶段二：PRD生成                             │
│  1. AI生成【02_PRD客户版】→ 客户确认功能范围                     │
│  2. 客户确认后，AI生成【03_PRD开发版】→ 包含技术细节             │
│  ⏱️ 参考时间：2-4小时                                          │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                     阶段三：Demo开发                            │
│  1. 使用【04_Demo快速启动指南】搭建框架                          │
│  2. 实现核心功能（P0级别）                                       │
│  3. 生成可运行Demo，发给客户体验                                 │
│  ⏱️ 参考时间：1-2天                                            │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                     阶段四：确认与迭代                          │
│  使用【05_迭代优化流程】                                        │
│  客户反馈 → 问题分类 → 优先级排序 → 修改实现 → 再次确认           │
│  ⏱️ 参考时间：每轮1-2小时                                      │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│                     阶段五：功能完善                            │
│  1. 核心功能确认后，添加P1、P2级功能                             │
│  2. 参考【06_非功能性检查清单】完善安全/性能/合规                │
│  3. 最终测试和上线准备                                          │
└──────────────────────────────────────────────────────────────┘
```

### 3.2 核心理念

| 理念 | 说明 | 解决的问题 |
|------|------|-----------|
| **需求先行** | 用结构化问卷确保需求完整、无歧义 | 避免来回确认浪费时间 |
| **双版本PRD** | 客户版侧重"能做什么"，开发版侧重"怎么做" | 客户和AI都能看懂 |
| **先Demo后迭代** | 快速可视化，降低沟通成本 | 让客户尽早看到效果 |
| **非功能性前置** | 合规、安全、性能在设计阶段就考虑 | 避免上线前才发现问题 |
| **经验持续积累** | 每个项目的问题和方案记录到知识库 | 避免重复踩相同的坑 |

### 3.3 角色分工

```
┌─────────────────────────────────────────────────────────┐
│  客户  ←→  你（项目负责人）  ←→  AI（开发助手）            │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  客户：提供需求、确认方案、验收成果                        │
│        ↕ 沟通                                           │
│  你：收集需求、转达反馈、协调进度、质量把控                 │
│        ↕ 协作                                           │
│  AI：生成PRD、编写代码、解决技术问题                       │
│                                                         │
│  【重要】客户不直接与AI对话，所有沟通通过你转发            │
└─────────────────────────────────────────────────────────┘
```

## 💡 五、最佳实践与经验总结

### 5.1 做对的事情（来自星见项目）

✅ **模块化开发**
- 一个模块完成再开始下一个，避免混乱
- 每个模块都有独立的完成总结文档

✅ **详细文档**
- 代码、配置、决策都有文档记录
- 文档与代码同步更新，保持一致性

✅ **统一规范**
- 云函数返回格式统一
- 代码风格、命名规范全部统一
- CSS变量集中定义

✅ **先核心后完善**
- 先跑通主流程，再补充边界情况
- P0功能确认后再做P1、P2

### 5.2 踩过的坑（必须避免）

⚠️ **SDK版本问题**
- ❌ 问题：wx-server-sdk 使用3.x版本导致失败
- ✅ 解决：必须用 ~2.6.3，不能用3.x

⚠️ **路径配置问题**
- ❌ 问题：vite.config.js的base和router的history不一致
- ✅ 解决：两者必须完全一致（如都是 /admin/）

⚠️ **权限配置问题**
- ❌ 问题：数据库权限设置不当导致读写失败
- ✅ 解决：读写分离，所有写操作通过云函数

⚠️ **非功能性遗漏**
- ❌ 问题：隐私弹窗、权限请求、弱网处理容易忘记
- ✅ 解决：使用【06_非功能性检查清单】逐项检查

⚠️ **CSS变量混乱**
- ❌ 问题：不同模块的样式变量不统一
- ✅ 解决：在App.vue中统一定义所有CSS变量

### 5.3 关键检查点

#### 需求阶段必确认
- [ ] 小程序类型（工具/电商/社区/服务等）
- [ ] 核心用户和使用场景
- [ ] 必须有的功能（P0级别）
- [ ] 是否需要后台管理
- [ ] 预算和时间约束

#### PRD阶段必确认
- [ ] 功能优先级排序（P0/P1/P2/P3）
- [ ] 页面数量和跳转关系
- [ ] 数据存储需求
- [ ] 是否需要微信支付
- [ ] 是否需要微信登录/手机号

#### Demo阶段必确认
- [ ] UI风格是否满意
- [ ] 核心操作流程是否顺畅
- [ ] 有无重大遗漏功能

#### 上线前必确认
- [ ] 【06_非功能性检查清单】全部过一遍
- [ ] 小程序名称、类目已确定
- [ ] 隐私协议、用户协议已准备
- [ ] 测试账号和测试数据已准备

### 5.4 工作流程最佳实践

**每次开发前**：
1. 阅读相关文档，理解现有实现
2. 检查【07_经验知识库】是否有类似问题
3. 明确本次开发的目标和范围

**每次修改后**：
1. 更新相关文档，保持同步
2. 运行检查清单，确保质量
3. 真机测试验证

**每个模块完成后**：
1. 写完成总结，记录关键点
2. 更新迭代记录
3. 客户确认后再进行下一步

**每次部署前**：
1. 过一遍非功能性检查清单
2. 备份当前版本
3. 准备回滚方案

## 🎯 七、项目特色与创新点

### 7.1 核心创新

**1. 双版本PRD机制**
- 传统：单一技术文档，客户看不懂或过于简化
- 创新：客户版（易懂）+ 开发版（精确），各取所需

**2. AI协作优化设计**
- 传统：文档给人看，AI重新理解
- 创新：文档直接针对AI设计，结构化、精确、完整

**3. 经验自动传承**
- 传统：经验在人脑中，难以传递
- 创新：结构化记录，AI自动读取，避免重复踩坑

**4. 非功能性前置**
- 传统：开发完成后才考虑合规、安全
- 创新：设计阶段就系统化检查，减少返工

### 7.2 工具价值

**对开发者的价值**：
- ✅ 减少80%的需求沟通时间
- ✅ 避免90%的常见技术坑
- ✅ 提高50%的开发效率

## 📝 十一、总结与展望

### 11.1 项目总结

**本项目的本质**：
- 不是一个具体的业务小程序
- 而是一套**标准化的开发流程和工具集**
- 目标是**提高AI协作开发的效率和质量**

**核心价值**：
1. **流程标准化**：从混乱到有序
2. **经验可传承**：从个人到团队
3. **质量可保证**：从经验到规范
4. **效率可提升**：从重复到复用

### 11.2 适用场景总结

**最适合**：
- 小型项目（1-3个月开发周期）
- AI协作开发（开发者+AI）
- 零技术客户（需要简化沟通）
- 需要快速迭代（MVP验证）

**不适合**：
- 大型复杂项目（需要更完善的项目管理）
- 纯人工开发（工具为AI优化）
- 高安全要求（需要更严格的规范）

### 11.3 使用建议

**给开发者**：
- 首次使用建议完整阅读所有文档
- 根据实际项目调整流程和模板
- 持续积累经验到知识库
- 定期备份数据

**给AI**：
- 每次对话开始先读取【07_经验知识库】
- 严格遵循【03_PRD开发版】的技术规范
- 参考【06_非功能性检查清单】保证质量
- 遇到问题先查询经验库再提问

### 11.4 未来展望

**工具演进方向**：
```
v1.0（当前）：标准化流程 + 文档模板 + 经验库
    ↓
v2.0：代码生成 + 自动检查 + 模板库
    ↓
v3.0：AI Agent + 全自动化 + 行业知识库
    ↓
v4.0：质量评分 + 持续优化 + 生态系统
```

**长期愿景**：
让AI能够像有经验的开发者一样，系统化、规范化地完成小程序开发，同时不断积累和传承经验，最终形成一个可持续进化的开发工具生态。
