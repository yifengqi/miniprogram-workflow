## 🎯 推荐方案：在系统终端操作

由于Cursor环境的限制，建议你在**系统终端（Terminal.app或iTerm）**中操作：

### 方式1：使用GitHub CLI（最简单）

```bash
# 1. 打开终端，进入项目目录
cd "/Users/yym/Documents/硬盘临时存储文件/星见 2026/星见小程序/miniprogram-workflow"

# 2. 使用gh CLI推送（会自动处理认证）
gh repo sync
```

如果`gh repo sync`不行，使用传统方式：

```bash
# 3. 普通git推送
git push origin main
```

如果提示需要认证，gh CLI会自动打开浏览器让你授权。

---

### 方式2：使用Personal Access Token

如果gh CLI有问题，可以使用GitHub的个人访问令牌：

**第1步：生成Token**
1. 打开浏览器访问：https://github.com/settings/tokens
2. 点击"Generate new token (classic)"
3. 勾选 `repo` 权限
4. 生成token并复制（只显示一次！）

**第2步：在终端推送**
```bash
cd "/Users/yym/Documents/硬盘临时存储文件/星见 2026/星见小程序/miniprogram-workflow"

# 推送时会提示输入用户名和密码
git push origin main

# Username: yifengqi
# Password: [粘贴你的token，不是GitHub密码]
```

**第3步：保存凭据（可选）**
```bash
# 保存凭据，下次不用再输入
git config --global credential.helper store
```

---

### 方式3：使用GitHub Desktop（最简单）

1. 打开GitHub Desktop应用
2. 添加仓库（如果还没添加）
   - File → Add Local Repository
   - 选择项目目录
3. 点击右上角的 **"Push origin"** 按钮
4. 完成！

## 💡 为什么在Cursor中推送困难？

```

├─ 沙箱环境限制网络访问
├─ 无法打开交互式认证界面
├─ VPN代理配置复杂
└─ 建议使用系统终端操作

系统终端的优势：
├─ 完整的网络访问权限
├─ 可以打开浏览器授权
├─ gh CLI工作正常
└─ 操作更简单直接
```

## 📝 推送成功后

代码推送到GitHub后，你就可以：

### 1. 部署到Vercel

**访问**：https://vercel.com

**步骤**：
1. 登录Vercel（使用GitHub账号）
2. 点击 "Add New Project"
3. 导入仓库：`yifengqi/miniprogram-workflow`
4. Vercel自动检测为Vite项目
5. 点击 "Deploy"
6. 等待部署完成（约2-3分钟）
7. 获得访问地址（如：`https://miniprogram-workflow.vercel.app`）

**公共表单地址**：
```
https://你的vercel域名/public-form
```

### 2. 分享给客户

部署成功后，就可以：
- 复制公共表单链接发给客户
- 生成二维码让客户扫码填写
