# 小程序开发工作流

一个基于 Vue 3 + Element Plus 的小程序开发标准化流程工具，支持 AI 辅助生成 PRD 文档。

## 功能特性

- **需求收集器** - 结构化表单收集客户需求，分步骤引导填写
- **公共需求表单** - 独立的客户需求收集页面，支持链接分享和二维码扫描
- **PRD 生成器** - 调用 AI 自动生成客户版和开发版 PRD
- **经验知识库** - 记录开发问题和解决方案，持续积累经验
- **检查清单** - 非功能性检查清单，确保项目合规、安全、高性能
- **客户提交管理** - 查看、导出、导入客户提交的需求
- **数据管理** - 支持数据导出/导入，本地存储不丢失

## 技术栈

- Vue 3 + Composition API
- Element Plus (深色主题)
- Pinia (状态管理)
- Vue Router
- Vite (构建工具)
- Vercel (部署)

## 快速开始

### 安装依赖

```bash
npm install
```

### 本地开发

```bash
npm run dev
```

访问 http://localhost:3000

### 构建生产版本

```bash
npm run build
```

## 配置 AI

1. 打开「设置」页面
2. 填写 API 端点（支持 OpenAI、Anthropic、DeepSeek 及各种中转站）
3. 填写 API Key
4. 选择模型
5. 点击「测试连接」确认配置正确

### 支持的 AI 服务

| 服务 | API 端点 | 模型示例 |
|------|----------|----------|
| OpenAI | `https://api.openai.com/v1/chat/completions` | gpt-4o, gpt-4o-mini |
| Anthropic | `https://api.anthropic.com/v1/messages` | claude-3-5-sonnet-20241022 |
| DeepSeek | `https://api.deepseek.com/v1/chat/completions` | deepseek-chat |
| 中转站 | 根据服务商提供的地址 | 根据服务商支持的模型 |

## 部署到 Vercel

1. Fork 或推送代码到 GitHub
2. 在 Vercel 中导入项目
3. 自动检测为 Vite 项目并部署
4. 可选：绑定自定义域名

## 数据存储

所有数据存储在浏览器 localStorage 中：

- 项目数据（需求、PRD）
- 经验知识库
- 检查清单进度
- AI 配置（API Key 仅本地存储，不上传）

**建议定期导出备份！**

## 使用方法

### 基本流程

1. **创建项目**：在首页创建新项目
2. **收集需求**：使用需求收集表单向客户收集信息
3. **生成PRD**：AI自动生成客户版和开发版PRD
4. **开始开发**：参考PRD和经验知识库进行开发
5. **检查质量**：使用检查清单确保项目质量

### 客户需求收集

**方式1：内部使用**
- 在系统内创建项目后，使用"需求收集"功能

**方式2：分享给客户**
1. 进入「客户提交」页面
2. 点击「复制表单链接」或「生成二维码」
3. 分享给客户填写（支持电脑和手机）
4. 客户提交后，在系统中查看并导入

**公共表单地址**：`https://你的域名/public-form`
