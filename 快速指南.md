> 5分钟快速了解项目和核心功能

---

## 📌 这是什么项目？

一个**标准化的小程序开发流程工具**，帮助开发者和AI协作，快速完成从需求收集到PRD生成的全过程。

**核心价值**：
- ✅ 结构化需求收集，避免遗漏
- ✅ AI自动生成双版本PRD（客户版+开发版）
- ✅ 积累经验知识库，避免重复踩坑
- ✅ 系统化检查清单，确保项目质量
- ✅ 支持客户独立填写需求（无需登录）

---

## 🎯 适用场景

**最适合**：
- 小型小程序项目（1-3个月开发周期）
- AI协作开发（开发者+AI助手）
- 客户零技术背景
- 需要快速迭代和验证

**不适合**：
- 大型复杂项目
- 纯人工开发团队

## ⚠️ 重要注意事项

### 1. API配置
❌ **常见错误**：API端点URL不完整
```
错误：https://api.example.com
正确：https://api.example.com/v1/chat/completions
```

✅ **正确做法**：
- 复制完整的API端点URL
- 确保API Key有效
- 先测试连接再使用

### 2. 客户需求收集
⭐ **新功能亮点**：
- 公共表单地址：`https://你的域名/public-form`
- 支持手机扫码填写
- 自动保存草稿
- 生成唯一提交ID

✅ **最佳实践**：
- 优先使用公共表单分享给客户
- 提交后在「客户提交」页面查看
- 一键导入到项目继续开发

### 3. 数据安全
⚠️ **重要提醒**：
- 所有数据存储在浏览器localStorage
- API Key仅本地存储，不上传
- **定期导出备份！**（设置→导出数据）

### 4. PRD生成
✅ **使用技巧**：
- 先生成客户版，给客户确认
- 客户确认后再生成开发版
- 两个版本都要保存

### 5. 经验知识库
💡 **关键作用**：
- AI每次开始项目前会自动读取
- 避免重复踩相同的坑
- 持续积累项目经验

### 6. 测试数据管理 ⭐ 重要
🔧 **解决开发痛点**：
- 开发时缺少测试数据导致各种边界问题
- 一键初始化完整测试数据
- 完善空状态处理，避免页面报错
- 详见：[开发环境测试数据方案.md](./标准化开发流程/开发环境测试数据方案.md)

### 7. 上线前检查
📋 **必做事项**：
- 过一遍「非功能性检查清单」
- 特别注意：隐私政策、用户协议
- 确保合规，避免审核被拒

### ⚠️ Git推送问题
**如遇到错误**：`Error in the HTTP2 framing layer`

**快速解决**：
```bash
# 方式1：手动在终端推送
cd 项目目录
git push origin main

# 方式2：使用GitHub Desktop
# 打开应用 → 选择仓库 → Push

# 方式3：切换SSH协议
git remote set-url origin git@github.com:用户名/仓库名.git
git push origin main
```

## 🆘 常见问题

**A**：进入「客户提交」页面，复制链接或生成二维码分享给客户

### Q5：Git推送失败怎么办？
